React

1. Introduction

1.1. What is it?

  • It is a JS library but it acts as a framework.
  • Used for building UI and front-end apps.

1.2. Why use it?

  • Makes front-end development faster and easier.
  • Uses self contained, independent components with their own state.
  • More interactive UIs
  • Uses Virtual DOM -> Makes manipulation tasks cheaper.
  • Uses JSX - Easily incorporate JS in markup.
  • Makes it easier to work with teams.

1.3. How to get started?

  • We use Create React App to setup a boilerplate code of our app.
  • It is CLI tool.
  • Automatically takes care of configurations using webpage.
  • Comes with dev server and hot reload.
  • npm run build wil come compile all the code for browser to read.

1.3.1. Steps to get started

  • Make sure NodeJS is installed (needed in order to use npm).
  • Use Create React App to make a boiler plate code. (GitHub Repo for Create React App). Use CLI command in the desired directory for this:
    • npx create-react-app my-app

1.4. What is State?

  • It is an object that determines how components renders and behaves.
  • We can also have application level state using a state manager like Redux or Reacts own context API.
state = { title: 'xxx', body: 'yyy', isFeatured: true }

1.5. Anatomy of a React App

1.5.1. package.json

Keeps track of all our project dependencies and scripts.

1.5.2. index.html

  • Sits inside the public directory of the project.
  • React is a single page application framework (a library really), i.e., everything runs from a single page and index.html is that page.
  • Everything is output onto
    div. Or we can say the main app component outputs here and everything is inside of that.
  • We can also use this page to include our CDNs in the header tag if we needs (eg. Bootstrap, fontAwesome, etc).

1.5.3. index.js

  • Sits in src directory of the project.
  • It produces the code that is output onto to the index.html
  • We import the libraries(react, react-dom, etc,.), css files, main App component, etc., the beginning of this file. Note: the App component is the main component that wraps around everything.
  • It also renders the App component into the root div of index.html

1.5.4. App.js

  • Sits in src directory of the project.
  • It is the component which is included by the index.js file into the root div of the index.html file.
  • First of all, we import React and React Component class into it.
  • We also import the app logo and all wide css App.css.
  • It is mostly just like any other react component.
  • It has a render method which returns JSX inside it. JSX looks like HTML.
      Note:
    1. render() is called the life cycle method.
    2. JSX is an easily way to write JS for output in the browser. We can use JS variables in it using curly braces { variableName } and we can't use class attribute in tags, instead we use the className attribute.

2. React Development

2.1. Components

2.1.1. Anatomy of a Component. (Class Component)

Class Post extends React.Component { state = { title: 'Post One', body: 'This is post one.' } render(){ return (
{ this.state.title } { this.state.body }
) } }

There are two types of components in ReactJS:

  1. Functional Component (a.k.a Stateless Components)
    • A Functional component is a function that takes props and returns JSX.
    • They do not have state or lifecycle methods.
    • It lacks the state object and, therefore, functional Component must be used when we don't need state in our component.
    • Functional components are easier to read, debug, and test. They offer performance benefits, decreased coupling, and greater reusability.
  2. Class Component
    • They are ES6 classes and do contain a state object.
    • They do contain the state object and the lifecyle methods.

2.1.2. Creating a new Component. (Class Component)

  1. Move to the src directory of the project and create a new sub-directory named components.
  2. Create a new file in it and name it with a .js extension at the end.
  3. Import the library: import React, { Component } from 'react';
  4. Add a class with the same name as the file which will extend the Component class. class ComponentName extends Component { }
  5. Add a render() method to the class which will return some JSX code. render(){ return ( //JSX Code. ); }
  6. Export the class at the end of the file. export default ComponentName;
  7. Go to App.js and import the new component to it. import ComponentName from './components/ComponentName';
  8. To embed the component into the App.js's render method, just do it like a custom tag with the Component's name.

2.3. Component Tree

  • In React we often need to include components inside other components.
  • A component can include multiple other components.
  • This can be visualized as a tree like structure.
  • This is what we call the Component Tree in React.

2.3.1. Sending Variables To Parent Component.

  • We can do this by creating a method in the parent component.
  • The method may take in some formal parameters in it's definition which can be sent via a child component.
    • customMethod = (var1) => { //Do some operation. }
  • After defining the method we send it the child component as a prop in it's component tag inside the parent component.
  • Child component can then access this method as a prop and then pass in variables by binding them to the prop including a this keyword.

2.5. HTTP Requests & Working With RESTful APIs

  • ReactJS is meant for front-end development which means in order for our application to be useful we have to use some sort of a bank-end in place (it may be NodeJS, FireBase, Laravel, etc,.).
  • Typically, front-end and back-end interact using REST APIs and they work over HTTP requests.
  • In order to make requests we can either go with JS' default way of using FetchAPI or you can use axios which is an HTTPS library (which we are going to use).

2.5.1. Implementing AXIOS library

  • To install axios, use:

    npm i axios

  • In order to fetch data using axios, we need to first import it into our component.

    import axios from 'axios';
  • We use axios in componentDidMount life-cycle method of the component.
2.5.1.1 Fetch Request Using Axios (Read)
componentDidMount() { const apiUrl = 'some.url/api/...'; axios.get(apiUrl) .then((response) => console.log(response.data)); }
2.5.1.2. Post Request Using Axios (Update)
const updateReq = { property1: value1, property2: value2, property3: value3, property4: value4 } axios.post(apiUrl, updateReq) .then(response => this.setState({ obj : [...this.state.obj, response.data] })); //Or you can do any other operation in "then" that you wish.
2.5.1.3. Delete Request Using Axios (Delete)

      import uuid from 'uuid';
